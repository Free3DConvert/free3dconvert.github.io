Workflow

How browser routes work

Check the file, then follow the route shown by the widget.

  1. 01

    Choose a route

    Open the converter page and check whether the file stays in the browser or needs manual handling.

  2. 02

    Upload the file

    Use drag and drop or the file picker. The widget checks the file type before it shows the next step.

  3. 03

    Continue with the right path

    Open formats can continue in the browser. Native source files are routed to manual handling.

  4. 04

    Read the result

    You will see whether the route can continue, needs manual handling, or is not available in the browser.

Support model

Browser or manual

The site shows which path applies before you start.

Open-format flow

Browser routes

  • Only show processing for supported open-format pairs.
  • Keep the status simple and direct.
  • Do not imply support for unsupported files.
Proprietary source

Manual routes

  • Use this for files like .c4d, .max, .ma, and .mb.
  • Show the limitation before processing starts.
  • Send users to contact instead of showing a fake success state.
